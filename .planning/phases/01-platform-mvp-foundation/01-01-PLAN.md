<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| developer->git/CI | Untrusted code changes cross into deployment pipeline. |
| CI->Vercel deployment | Build artifacts and environment config cross into production hosting. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | `.github/workflows/ci.yml` | mitigate | Enforce mandatory lint/typecheck/build gate before merge/deploy. |
| T-01-02 | Repudiation | deployment workflow | mitigate | Keep auditable CI runs tied to commit SHA and PR history. |
| T-01-03 | Denial of Service | misconfigured deployment defaults | mitigate | Use deterministic build/runtime config and verify via build on each change. |
| T-01-SC | Tampering | npm installs in scaffold commands | mitigate | Use only packages from approved stack and lock dependency versions through lockfile review in execution. |
</threat_model>
