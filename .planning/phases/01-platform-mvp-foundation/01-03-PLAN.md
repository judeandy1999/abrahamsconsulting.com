## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| editor content changes->validation pipeline | Human-authored content crosses into production build inputs. |
| content modules->public page rendering | Structured content is transformed into public output. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-07 | Tampering | `src/content/*.ts` | mitigate | Enforce schema validation in local and CI paths before build/deploy. |
| T-01-08 | Denial of Service | malformed content causing build failures late | mitigate | Shift validation to explicit script and prebuild hook for early failure detection. |
| T-01-09 | Elevation of Privilege | executable logic in content modules | mitigate | Keep content files as typed data only; route logic remains in application code. |
| T-01-SC | Tampering | dependency usage in validator | mitigate | Restrict validator dependencies to approved stack and commit lockfile changes intentionally. |
</threat_model>
